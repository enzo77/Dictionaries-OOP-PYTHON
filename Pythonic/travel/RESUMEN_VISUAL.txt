⚙️  FLUJO DE EJECUCIÓN
═════════════════════════════════════════════════════════════════════════════

with TravelDatabase("travel.db") as connection:
    │
    ├─ 1️⃣  __enter__() se ejecuta:
    │   ├─ Abre conexión a SQLite
    │   ├─ Crea tabla trips si no existe
    │   └─ Retorna la conexión
    │
    ├─ 2️⃣  Código del with block:
    │   ├─ repository = SQLiteTravelRepository(connection)
    │   ├─ repository.add_trip(...)
    │   ├─ repository.find_by_country(...)
    │   └─ repository.average_price()
    │
    └─ 3️⃣  __exit__() se ejecuta (SIEMPRE):
        ├─ ¿Hubo excepción?
        │   ├─ NO  → COMMIT (guardar cambios)
        │   └─ SÍ → ROLLBACK (deshacer cambios)
        └─ Cierra la conexión

🎯 CONCEPTOS CLAVE
═════════════════════════════════════════════════════════════════════════════

1. CONTEXT MANAGER
   • __enter__(): Se ejecuta al entrar en el bloque with
   • __exit__(): Se ejecuta siempre al salir (con o sin errores)
   • Garantiza limpieza automática de recursos
   • Maneja COMMIT/ROLLBACK automáticamente

2. MIXINS
   • Clases que añaden funcionalidad opcional
   • Se heredan junto a otras clases (herencia múltiple)
   • Permiten composición horizontal de comportamiento
   • Evitan jerarquías de herencia profundas

3. TRANSACCIONES
   • COMMIT: Guarda los cambios en la BD (éxito)
   • ROLLBACK: Deshace los cambios (error)
   • ACID: Atomicity, Consistency, Isolation, Durability

4. SEGURIDAD SQL
   • Usa ? como placeholders en lugar de f-strings
   • Protege contra SQL Injection
   • Los parámetros se escapan automáticamente

# Usar el context manager

    repo = SQLiteTravelRepository(connection)
    
    # Agregar datos
    repo.add_trip("Tokyo", "Japan", 7, 1500.0)
    repo.add_trip("London", "UK", 4, 800.0)
    
    # Buscar
